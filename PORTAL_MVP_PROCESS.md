# Портал личных кабинетов и конструктор чат-ботов (MVP)

## 1. Цель проекта

Создать веб-портал с личными кабинетами, в котором пользователи могут:
- создавать и настраивать Telegram-ботов;
- визуально описывать бизнес-процессы бота в формате BPMN (блоки/стрелки);
- публиковать процесс в 1 клик;
- запускать сценарий в работу сразу после публикации.

## 2. Роли пользователей

- Админ
- Менеджер
- Агент поддержки

## 3. Авторизация

- Логин/пароль
- OAuth: Google, Telegram
- 2FA на этапе MVP не требуется

## 4. Монетизация

- Подписка
- Тарифы
- Лимиты по тарифу:
  - количество ботов;
  - количество сообщений;
  - количество интеграций

## 5. Каналы и интеграции

### Каналы v1

- Telegram

### Интеграции v1 (приоритет для реализации)

- Webhook
- OpenAI
- 1С
- CRM
- Google Sheets

## 6. Функциональный объем MVP

- Личный кабинет пользователя
- Управление ролями в рамках аккаунта/организации
- Создание и настройка Telegram-бота
- BPMN-конструктор процессов (блоки/связи)
- Валидация процесса перед публикацией
- Публикация сценария в 1 клик
- Исполнение опубликованного сценария в runtime
- Журнал событий (действия пользователя, публикации, ошибки, runtime-события)
- Базовая аналитика

## 7. Архитектурные принципы

- Модульная архитектура (сервисный подход)
- Отдельный слой адаптеров каналов (начать с Telegram)
- Отдельный слой интеграций (коннекторы)
- Версионирование бизнес-процессов
- Публикация активной версии без остановки сервиса
- Закладка на горизонтальное масштабирование

## 8. Хранение данных

- На этапе MVP: локальная база данных на текущем сервере
- Обязательно использовать миграции схемы БД
- Структуру БД проектировать с учетом дальнейшей миграции на отдельный сервер/кластер

## 11. UX-потоки (MVP)

1. Регистрация/вход -> создание организации -> выбор тарифа (или trial)
2. Создание бота -> подключение Telegram -> статус "черновик"
3. Проектирование процесса в BPMN-редакторе -> валидация -> сохранение
4. Публикация в 1 клик -> создание версии -> активация runtime
5. Мониторинг через журнал событий

## 12. Roadmap реализации

### Этап 1. Основа платформы (1-2 недели)

- Авторизация и роли
- Организация/аккаунт
- Базовый каркас личного кабинета
- Базовая модель тарифов/лимитов

### Этап 2. Telegram и core бота (1-2 недели)

- Создание бота
- Подключение Telegram webhook
- Отправка/прием сообщений

### Этап 3. BPMN-конструктор (2-3 недели)

- Визуальный редактор
- Сохранение схем
- Валидация процесса
- Версионирование

### Этап 4. Runtime и публикация (1-2 недели)

- Запуск опубликованной версии
- Переключение активной версии процесса

### Этап 5. Интеграции и журнал (1-2 недели)

- Webhook + OpenAI (минимум)
- Журнал событий
- Базовая аналитика

### Этап 6. Стабилизация

- Тестирование
- Оптимизация
- Бэкапы
- Подготовка миграции БД

## 13. Критерии готовности MVP

- Пользователь может зарегистрироваться и войти в личный кабинет
- Пользователь с нужной ролью может создать Telegram-бота
- Процесс бота можно собрать визуально, сохранить и опубликовать
- После публикации бот исполняет актуальный процесс
- Действия и ошибки фиксируются в журнале
- Ограничения по тарифу корректно применяются
